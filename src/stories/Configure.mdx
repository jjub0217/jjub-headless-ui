# jjub-headless-ui

Headless UI components with built-in accessibility. Unstyled, customizable, WAI-ARIA compliant.

## What is Headless UI?

Headless UI components provide **logic and accessibility** without any visual styles.
You bring your own styles via `className` or `data-state` attributes.

- **Zero styles** — no colors, fonts, or spacing included
- **Full accessibility** — WAI-ARIA roles, keyboard navigation, focus management
- **Customizable** — style with CSS, Tailwind, CSS-in-JS, or any approach
- **Zero dependencies** — only React as peer dependency
- **TypeScript-first** — full type definitions included

## Installation

```bash
npm install jjub-headless-ui
```

## Quick Start

```tsx
import { Tabs } from 'jjub-headless-ui'

function App() {
  return (
    <Tabs.Root defaultValue="tab1">
      <Tabs.List>
        <Tabs.Tab value="tab1">Tab 1</Tabs.Tab>
        <Tabs.Tab value="tab2">Tab 2</Tabs.Tab>
      </Tabs.List>
      <Tabs.Panel value="tab1">Content 1</Tabs.Panel>
      <Tabs.Panel value="tab2">Content 2</Tabs.Panel>
    </Tabs.Root>
  )
}
```

## Styling with data-state

All components expose `data-state` attributes for CSS styling:

```css
[role="tab"][data-state="active"] {
  font-weight: bold;
}
[role="tab"][data-state="inactive"] {
  color: gray;
}
```
